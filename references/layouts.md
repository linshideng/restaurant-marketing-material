# 布局参考

布局元数据位于 `assets/layouts/manifest.json`。`layout_preference` 只作为同分时的 tie-breaker，主要选择依据始终是物料类型、比例、是否有二维码、是否有真实菜品图。

| Layout | 名称 | 适用场景 |
|---|---|---|
| A | 大字冲击型 | 海报、KT 板、展架、传单；标题强、远距离可见 |
| B | 场景沉浸型 | 全屏氛围背景；适合餐厅烟火气和活动主题 |
| C | 扫码转化型 | 台卡、传单；二维码是主视觉 |
| D | 菜品英雄型 | 有真实菜品图时使用；菜品图作为参考输入 AI 模型 |
| E | 活动信息型 | 信息较多的传单或 KT 板 |
| F | 简约台卡型 | 小画幅、大二维码、短 CTA |

不要把二维码放进 AI 生成 prompt。QR 由后合成管线填充。AI 模型可以渲染用户提供的展示文字、价格、CTA 等。
